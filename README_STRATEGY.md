# AgentKit（エージェントキット） - 戦略ドキュメント

## プロダクト概要（逆引きPRD）

### ターゲットユーザー
- フリーランスのデザイナー・エンジニア・コンサルタント（定型業務を自動化したい人）
- 小規模事業主（クライアント対応・請求管理・レポート作成に時間を取られている人）
- ノーコードツール活用者（Zapierのような自動化ツールにAI判断を加えたい人）

### 解決する課題
- フリーランスはクライアントオンボーディング、請求リマインダー、進捗レポートなどの定型業務に多くの時間を費やしている
- 既存の自動化ツール（Zapier等）にはAI判断ノードがなく、「条件によって対応を変える」といった柔軟な自動化ができない
- プログラミングなしでAIエージェントを構築できるツールが日本語対応で存在しない

### ソリューション
ノーコードのビジュアルエディターでAIエージェントを構築するプラットフォーム。トリガー → AI判断 → アクション → 条件分岐のワークフローステップを組み合わせ、フリーランス業務を自動化する。テンプレートから即座に稼働開始可能。

### コア機能（実装済み）
- **ノーコードビルダー**: ドラッグ＆ドロップでワークフローステップを定義
- **4種類のステップタイプ**: トリガー（手動/スケジュール/Webhook）、AI判断、アクション、条件分岐
- **AI Brain設定**: 自然言語でAIの性格・判断基準をカスタマイズ
- **テンプレートギャラリー**: クライアントオンボーディング、請求リマインダー、進捗レポート、MTGスケジューラー、メール自動分類、コンテンツスケジューラー
- **実行シミュレーション**: エージェントのテスト実行（成功/失敗のモック、AI判断サンプル出力）
- **実行ログ管理**: 全実行結果の記録・可視化（ステータス、ステップ完了数、AI判断内容、エラーメッセージ）
- **エージェントステータス管理**: アクティブ / 一時停止 / 下書きのライフサイクル
- **デモデータ**: 初回アクセス時にサンプルエージェント3体と実行ログ5件を自動生成

## Stripeマネタイズ戦略

### 現状
Stripe未実装。全機能が無料で利用可能。データはlocalStorageに保存。実行はシミュレーション（モック）のみで実際のAPI連携は未実装。

### 無料プラン制限（推奨）
- エージェント数: **2体まで**
- ワークフローステップ: **エージェントあたり3ステップまで**
- 実行回数: **月10回まで**（シミュレーション含む）
- テンプレート: 基本2種類のみ（クライアントオンボーディング、請求リマインダー）
- 実行ログ保持: **直近30件まで**

### プレミアム機能（推奨）
- エージェント数: **無制限**
- ワークフローステップ: **無制限**
- 実行回数: **無制限**
- 全テンプレートアクセス
- 実際のAPI連携（Gmail、Slack、Notion、Googleカレンダー）
- Webhook受信（外部サービスからのトリガー）
- カスタムAI Brain（GPT-4/Claude連携の実際のAI判断）
- 実行スケジューリング（cron式の定期実行）
- チーム共有・コラボレーション

### 推奨価格
- **Free**: 0円（2エージェント、月10回、シミュレーションのみ）
- **Pro**: 1,980円/月（無制限エージェント、実API連携、月500回実行）
- **Business**: 4,980円/月（無制限実行、Webhook、カスタムAI、チーム管理）

### ペイウォール配置
- エージェント作成画面: 3体目の作成時に「Proプランでエージェント無制限」モーダル
- 実行ボタン: 月11回目の実行時に「今月の実行枠を使い切りました」表示
- テンプレート選択: プレミアムテンプレートにロックアイコン
- API連携設定: 「Pro: 実際のGmail/Slack連携を有効化」CTA
- ステップ追加: 4ステップ目の追加時に「Proで複雑なワークフローを構築」誘導

## リテンション施策

### メール通知トリガー（実装後）
- エージェント作成完了時: 「最初のテスト実行を試しましょう」ガイドメール
- 実行失敗時: 「エージェントXの実行でエラーが発生しました」即時アラート
- 週次サマリー: 「今週の実行結果: 成功X件 / 失敗Y件 / 承認待ちZ件」レポート
- 7日間未実行: 「エージェントが待機中です。今すぐ実行して業務を自動化しましょう」リマインダー
- 月初: 「先月の自動化で推定X時間を節約しました」効果レポート

### データ蓄積フック
- エージェント設定の蓄積（ワークフロー定義 + AI Brain設定が個人の業務プロセスを反映するため移行困難）
- 実行ログの蓄積（過去の実行結果・AI判断のパターンが学習データとして蓄積）
- テンプレートのカスタマイズ履歴（汎用テンプレートが自分専用に最適化される過程）

### 習慣形成
- ダッシュボードの「今日のエージェント稼働状況」リアルタイムウィジェット
- 「自動化時間節約」メトリクス表示（実行回数 x 推定手動作業時間 = 節約時間）
- 承認待ちアクションの「今すぐ承認」通知バッジ（日常的なチェック習慣）

## 不足機能（Quick Wins）

### 1. 実際のAPI連携（Gmail / Slack / Notion）
**説明**: 現在の実行はシミュレーション（モック）のみ。OAuth認証を通じてGmail送信、Slack通知、Notion DB更新などの実際のアクションを実行できるようにする。
**効果**: 「本当に業務が自動化される」という本質的な価値の実現。シミュレーションのみでは継続利用の動機が弱い。Pro課金の最大の根拠
**工数**: 大（OAuth2.0フロー + 各サービスAPI連携 + エラーハンドリング）

### 2. ワークフロービジュアルエディター
**説明**: 現在のステップ定義はリスト形式だが、ステップ間の接続をフローチャート形式で視覚的に表示・編集できるビジュアルエディター。条件分岐の分岐先を矢印で接続する直感的なUI。
**効果**: 「ノーコード」の訴求力を大幅に強化。フローチャートUIはランディングページのスクリーンショット素材としても有効
**工数**: 中〜大（React Flow等のライブラリ利用で中程度に抑えられる）

### 3. 実行スケジューリング（cron連携）
**説明**: 現在は手動実行のみ。「毎週月曜9:00」「毎月末」「毎日18:00」などのスケジュール設定で自動実行する機能。Vercel Cronまたは外部スケジューラー連携。
**効果**: 「設定したら放置でOK」の自動化体験を実現。請求リマインダーや週次レポートの自動送信が現実的に
**工数**: 中（Vercel Cron Jobs / Supabase Edge Functions + スケジュール管理UI）
